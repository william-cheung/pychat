
1. 客户端可以用户名、密码登陆进入软件（命令：./client.py -l USERNAME ）	
2. 客户端可以注册新用户 （命令：./client.py -r ）			 
3. 支持多个频道，如世界频道，21game频道或普通用户创建的频道（房间）
4. 用户可以查看房间列表、创建房间或者进入某个房间聊天	 
5. 不支持私聊。就目前的设计要支持私聊，客户端需要能接受除聊天消息以外的异步消息， 
   客户端服务器都需要维护更多的状态，实现比较繁琐 		
6. 关于21 点游戏										
     a. 在每分钟正点如12:31:00 服务器会随机生成4个1到10内的数，发布在房间21game 内，
        在房间的所有人都可以看到									
     b. 玩家可以用+,-,*,/和括号()，让4个数的计算结果尽量接近于21，但不能超过21;	
     c. 玩家可以用聊天的方式，向服务端提供回答，如: \21game (1+2)*(3+4)。只接受任意
        一个玩家第一次的回答，且此回答其他人看不到
     d. 服务端发布问题后，30 秒之内，如果有玩家的回答，刚好等于21，那么此玩家获胜，
        否则计时结束时，计算结果最大的玩家获胜，如果有相同的答案，取第一个回答者;
7. 应用层协议，请参见protocol.txt
